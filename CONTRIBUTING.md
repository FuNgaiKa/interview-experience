# Contributing

欢迎补充面试记录、技术笔记和复盘总结。为了保证仓库长期可维护，请遵循以下规范。

## 内容分类

- 面试记录放在 `interviews/` 目录下，按公司或组织分类。
- 技术笔记放在 `notes/` 目录下，按知识领域分类。
- 写作规范、模板和维护说明放在 `docs/` 目录下。

## 文件命名

- 保留中文文件名，优先使用清晰、可搜索的标题。
- 面试记录建议使用 `公司-部门-轮次.md` 或现有风格，例如 `腾讯云一面.md`。
- 技术笔记建议使用明确主题，例如 `数据库acid.md`、`分布式锁.md`。

## 面试记录建议结构

```markdown
# 公司/部门/轮次

## 基本信息

- 时间：
- 时长：
- 岗位：
- 形式：
- 结果：

## 面试问题

## 算法题

## 复盘总结

## 待补充知识点
```

## 技术笔记建议结构

```markdown
# 主题

## 核心概念

## 高频问题

## 关键结论

## 常见误区

## 参考资料
```

## 维护原则

- 保留真实复盘，不为了完整性编造不存在的信息。
- 不确定的答案要显式标记，后续再补充验证。
- 优先补充高频、基础、可复用的问题。
- 保持内容简洁，避免无关扩展。
