# 写作指南

本指南用于统一面试记录和技术笔记的写作格式。

## 面试记录

面试记录应该优先保留真实问题、回答情况和复盘结论。推荐结构如下：

```markdown
# 公司/部门/轮次

## 基本信息

- 时间：
- 时长：
- 岗位：
- 形式：
- 结果：

## 面试问题

1. 问题一
2. 问题二
3. 问题三

## 算法题

- 题目：
- 思路：
- 复杂度：
- 复盘：

## 项目问题

## 反问

## 复盘总结

## 待补充知识点
```

## 技术笔记

技术笔记应该围绕一个明确主题展开，重点服务面试复习和知识查漏补缺。推荐结构如下：

```markdown
# 主题

## 核心概念

## 高频问题

## 关键结论

## 常见误区

## 参考资料
```

## 写作原则

- 先记录原始问题，再补充答案和复盘。
- 区分事实、个人理解和待验证内容。
- 标题要具体，方便后续搜索。
- 不确定的内容使用 `TODO` 或 `待确认` 标记。
- 引用外部资料时保留链接。
